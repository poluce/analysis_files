PlantUML中的**注释**有几种写法：

## 注释语法

### 1. 单行注释 `'`（单引号）

```plantuml
' 这是单行注释
class MyClass

' 定义颜色主题
skinparam class {
    BackgroundColor #E3F2FD
}
```

### 2. 多行注释 `/' ... '/`

```plantuml
/'
这是多行注释
可以写很多行
用于详细说明
'/

class MyClass {
    /' 
    这个方法用于处理数据
    参数说明：...
    '/
    +processData()
}
```

### 3. 图表中的注释/说明文字

#### 使用 `note` 关键字

```plantuml
@startuml
class ThermalCurve {
    +addPoint()
}

' 右侧注释
note right of ThermalCurve
    这个类用于管理
    热分析曲线数据
end note

' 左侧注释
note left of ThermalCurve : 支持多种仪器类型

' 顶部注释
note top of ThermalCurve
    重要提示：
    数据点必须按时间排序
end note

' 底部注释
note bottom of ThermalCurve : 实现了观察者模式
@enduml
```

#### 连接到关系的注释

```plantuml
@startuml
class A
class B

A --> B : 使用
note on link
    这个依赖关系
    表示A使用B的功能
end note
@enduml
```

## 完整示例

```plantuml
@startuml 热分析系统架构

' ====================================
' 这是文件头注释
' 作者: 你的名字
' 日期: 2025-11-03
' 说明: 热分析软件的领域模型
' ====================================

/' 
多行注释示例：
这个图展示了热分析系统的核心类关系
包括项目、曲线、数据点等核心概念
'/

' 定义颜色主题
skinparam class {
    BackgroundColor<<Entity>> #FFF3E0
    BorderColor<<Entity>> #F57C00
}

' 核心实体类
class ThermalProject <<Entity>> {
    -projectName: QString
    -createDate: QDateTime
    +addCurve(): void
    +removeCurve(): void
}

note right of ThermalProject
    项目是顶层聚合根
    管理多条热分析曲线
end note

class ThermalCurve <<Entity>> {
    -curveName: QString
    -instrumentType: InstrumentType
    +addDataPoint(): void
}

note left of ThermalCurve : 每条曲线对应一次实验

class ThermalDataPoint {
    -temperature: double
    -time: double
    -signal: double
}

' 定义关系
ThermalProject o-- "many" ThermalCurve
note on link
    聚合关系
    曲线可以独立存在
end note

ThermalCurve *-- "many" ThermalDataPoint
note on link : 组合关系，数据点依附于曲线

' 枚举类型
enum InstrumentType {
    DSC
    TGA
    DTA
}

ThermalCurve --> InstrumentType

' 添加图例说明
legend right
    |= 符号 |= 含义 |
    | o-- | 聚合（弱拥有） |
    | *-- | 组合（强拥有） |
    | --> | 依赖 |
endlegend

@enduml
```

## 在Qt项目中添加注释

如果你要在Qt项目文档中使用PlantUML，可以这样组织：

```cpp
/**
 * @file ThermalCurve.h
 * @brief 热分析曲线类
 * 
 * @startuml
 * ' UML类图
 * class ThermalCurve {
 *     +addDataPoint()
 * }
 * @enduml
 */
class ThermalCurve {
    // C++代码
};
```

## 快捷输入方式

### 在不同编辑器中：

**VS Code / Qt Creator:**

- 单引号 `'` 直接输入
- 多行注释 `/'` + `'/`

**键盘输入：**

- 单引号：键盘上数字1左边的按键（中文输入法下可能需要切换到英文）
- `/`：问号键（中文下可能是顿号）

### 中文输入法下的注意事项

```plantuml
' 正确：英文单引号
' 错误：中文单引号（全角）

/' 正确 '/
/＇ 错误 ＇/  ← 这是全角符号，不会生效
```

## 推荐的注释风格

```plantuml
@startuml
' ============================================
' 文件信息区
' ============================================
' 文件名: domain_model.puml
' 作者: Zhang San
' 日期: 2025-11-03
' 版本: 1.0

' ============================================
' 颜色定义区
' ============================================
skinparam class {
    BackgroundColor #E8F5E9
}

' ============================================
' 类定义区
' ============================================
class MyClass {
    ' 属性定义
    -field1: int
    
    ' 方法定义
    +method1(): void
}

' ============================================
' 关系定义区
' ============================================
ClassA --> ClassB

@enduml
```

这样的注释风格清晰明了，方便维护。对于追求高级开发的你来说，良好的文档和注释习惯非常重要！
